## What this is

A Jekyll blog (Korean-language) by 김수명, served at https://sumyeong.kim (custom domain via `CNAME`, also reachable at lifespank.github.io). Built on the **jekyll-theme-chirpy** theme (gem-based; pinned to `~> 7.5` in `Gemfile`). The repo's purpose is content: most changes are new posts under `_posts/`, not code.

## Common commands

```sh
bundle install              # first-time setup / after Gemfile changes
bundle exec jekyll serve    # local preview at http://localhost:4000 with live reload
bundle exec jekyll build    # one-shot build into _site/
```

`_site/` is the generated output — never hand-edit it. Windows file watching uses the `wdm` gem (already in `Gemfile`).

## Authoring posts

Posts live in `_posts/` and **must** be named `YYYY-MM-DD-<title>.md` (`.markdown` also works). Required front matter pattern, mirroring existing posts:

```yaml
---
title: "제목"
date: 2026-05-03 15:53 +0900     # always +0900 (Asia/Seoul, set in _config.yml)
categories: ['윈도우 드라이버']    # use one of the established categories
tags: 윈도우 드라이버              # space-separated
---
```

`layout: post`, `comments: true`, `toc: true`, and the `/posts/:title/` permalink are applied automatically by the `defaults` block in `_config.yml` — do **not** repeat them in post front matter.

### Categories

Categories pages are auto-generated by the `jekyll-archives` plugin at `/categories/<name>/`. The established set is:

- 프로그래밍 문제
- 프로젝트
- 안드로이드
- 알고리즘 및 자료구조
- 윈도우 드라이버
- 여러가지

Adding a new category just creates a new `/categories/<new>/` page automatically — no nav file to update. The top-nav "Categories" tab (`_tabs/categories.md`) lists them all.

### Cross-linking, images, and admonitions

- Link to other posts with Liquid: `{% post_url 2026-02-16-예물 시계 구매하기 %}` (filename without extension, exact spaces preserved).
- Images go in `/images/` at repo root and are referenced via `{{ "/images/foo.png" | relative_url }}` — do not hardcode the site URL.
- Note callouts use GitHub-flavored admonitions (Chirpy renders them natively as styled blocks):
  ```markdown
  > [!NOTE]
  > 이 항목을 읽기 전에...
  ```
  Other supported types: `[!TIP]`, `[!IMPORTANT]`, `[!WARNING]`, `[!CAUTION]`. Do **not** use the legacy `{: .notice--info}` kramdown block — it was a minimal-mistakes idiom and won't render in Chirpy.

## Site architecture quirks worth knowing

- **Navigation** is driven by `_tabs/*.md`. Each tab file's front matter sets `icon`, `order`, and (for archive views) `layout: categories|tags|archives`. Add a new tab by dropping a new `.md` file in there.
- **Comments, search, locale:** Disqus shortname is `sumyeongkim` (set in `_config.yml`), Chirpy's built-in lunr-style search is on by default, `lang: ko` enables the Korean locale via the theme's bundled `_data/locales/ko.yml`.
- **Plugins** included via the `jekyll-theme-chirpy` gem (jekyll-archives, jekyll-paginate, jekyll-redirect-from, jekyll-seo-tag, jekyll-sitemap, etc.) — no need to declare them in `Gemfile` or `_config.yml`. The site does not use `github-pages` gem.
- **Top-level `*.html` / `*.md` pages** (`404.html`, `love42_privacy.html`, `love42_terms.md`, `naver*.html`, `robots.txt`, `feed.xml`) are intentional — they're either custom pages, app legal pages, or webmaster verification files. Don't treat them as stray.
